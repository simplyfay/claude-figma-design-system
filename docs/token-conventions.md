# Token Naming Conventions

> This file is auto-updated by `/build-token-system`. Add permanent notes below the generated sections.

## Architecture

Three-tier token system:

```
Primitive  →  Semantic  →  Component
color.blue.500  →  color.interactive.primary  →  button.background.default
```

### Tier 1: Primitives

Raw values. Named by hue + numeric scale step (50–900). No semantic meaning.
Never used directly in components — only referenced by semantic tokens.

### Tier 2: Semantic Tokens

Reference primitives. Carry design intent. Define light/dark modes.
Format: `{category}.{role}.{variant}`

Categories: `color`, `typography`, `spacing`, `radius`, `shadow`

### Tier 3: Component Tokens

Scoped to a single component. Reference semantic tokens.
Format: `{component}.{property}.{state}`

## Mode Strategy

- Modes are defined at the semantic tier only
- Light mode is the default (`:root`)
- Dark mode toggled via `[data-theme="dark"]` on `<html>`
- Primitives have no modes — they are static
